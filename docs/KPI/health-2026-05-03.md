## 🔴 Per-Lib Profile (`beast.profile`)

Single-run `beast.profile` capture, recording uptime **368.524 ms**.

> ⚠️ The recording uptime (369 ms) is much longer than the `--startuptime`
> total (46 ms). This is because `auto_dump_on_quit` fires on `VimLeavePre`,
> which the `timer_start(0, qa!)` in the autocmd doesn't dispatch until the
> event loop drains other pending work. **The function self-times are still
> correct** — only the wall-clock window of the recording is inflated. Fix
> in a future iteration: have `--headless` exits trigger the dump
> immediately rather than via a deferred timer.

- `beast.libs.key.highlights`

🟢 Confirmed intentional. Each second require is fast (~2 ms TOTAL), so
